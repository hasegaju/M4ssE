# SCAN LOG 実運用版 — バーコード読取・蓄積

iPhone + Bluetoothリーダー(HID) / カメラで動く、完全オフラインのバーコード収集ツール。
デモ版からフォークした**実運用版**。最大の違いはデータの永続化:
**スキャンのたびに端末内(IndexedDB)へ自動保存され、アプリを閉じても・再起動しても消えません。**

## 運用サイクル

```
場所・担当を入力 → スキャンして溜める → CSV書き出し → クリア確認 → 次の作業へ
```

- CSV書き出し後に「クリアしますか?」と確認が出るので、OKすれば次の棚卸しをまっさらから開始
- CSVはPCに送って(AirDrop / メール / 共有)Excel等で集計する前提(バッチ型ハンディターミナル運用)

## デモ版との違い

| | デモ版 | 実運用版(このリポジトリ) |
|---|---|---|
| データ | メモリのみ。閉じると消える | IndexedDBに自動保存。閉じても残る |
| 初期データ | サンプル入り | なし(空から開始) |
| 日時 | 時刻のみ | 年月日+時刻(複数日運用対応) |
| CSV | コード/種別/数量/時刻 | +場所/担当 列、書き出し後クリア確認 |
| キャッシュ名 | scanlog-v* | scanops-v*(共存可能。両方ホーム画面に置ける) |

## 設置手順(一度だけ)

1. 6ファイル+READMEを新しいGitHubリポジトリにアップロード
2. Settings → Pages → Branch: main / (root) → Save
3. 発行されたURLをiPhoneのSafariで開く(**HTTPS必須** — GitHub Pagesなら自動)
4. 共有 →「ホーム画面に追加」
5. 機内モードでホーム画面から起動確認、カメラも一度起動して許可を済ませる

## 機能

- HIDリーダー(Bluetooth・サフィックスEnter) / カメラ(📷)両対応、併用可
- 数量入力モード ON: 読取→数量パッド→確定 / OFF: 即登録・重複自動+1
- JAN-13/JAN-8 チェックディジット検証(緑バッジ)、その他は CODE 扱い
- 同一コードは数量に集約(CSVは吐いた時点で集計済み)
- ⌨️ ボタンで手入力キーボードの呼び出し(通常はソフトキーボード抑止)
- 🔊/🔇 読取音の切替(マナーモード中は鳴りません)

## 注意事項

- **場所・担当欄について**: Bluetoothリーダー接続中はiOSがソフトキーボードを出さないため、
  これらの欄を手入力する時はリーダーのキーボード呼び出しボタンを使うか、一時的にリーダーをOFFに
- IndexedDBは端末内保存であり万能ではありません(iOSのストレージ逼迫時に削除される可能性はゼロではない)。
  **作業の区切りごとにCSVを書き出す運用**を守ること。書き出したCSVが正本です
- 蓄積データはこの端末のこのアプリ内にのみ存在します。端末をまたぐ共有はCSV経由で

## 更新の反映

index.html を修正したら sw.js の `VERSION` を `scanops-v2` 等に上げる →
オンライン状態でアプリを開き直すとキャッシュが更新される。
反映確認はカメラ画面右下のバージョン表示(現在: o2)。
